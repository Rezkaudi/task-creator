FIGMA JSON SCHEMA INSTRUCTIONS

1. Valid Node Types: You must ONLY use these node types:
   - FRAME (main container, supports auto-layout)
   - GROUP (visual grouping without auto-layout)
   - RECTANGLE (shapes, backgrounds, icon containers)
   - TEXT (all text content)
   - ELLIPSE (circles, ovals, arcs, donut/pie shapes)
   - VECTOR (for custom shapes ONLY — NOT for icons)
   - LINE (horizontal/vertical lines, dividers)
   - POLYGON (triangles, hexagons, octagons — any regular polygon)
   - STAR (star shapes)
   - COMPONENT (reusable design component)
   - COMPONENT_SET (container for component variants)
   - INSTANCE (instance of a component)
   - BOOLEAN_OPERATION (union, subtract, intersect, exclude of shapes)
   - SECTION (canvas-level grouping container for organizing frames)

   Do NOT invent types like "BUTTON", "INPUT", "CARD", "ICON", etc.
   - A button = FRAME or RECTANGLE containing a TEXT node
   - An input field = FRAME with TEXT inside
   - A card = FRAME with children
   - An icon = RECTANGLE with IMAGE fill from Iconify API (see icon instructions)

2. Color Values: All color values (r, g, b) must be between 0 and 1 (NOT 0-255).
   - White: { "r": 1, "g": 1, "b": 1 }
   - Black: { "r": 0, "g": 0, "b": 0 }
   - Blue: { "r": 0.09, "g": 0.46, "b": 0.95 }

3. Required Properties for Each Node:

   All Nodes:
   - name (string)
   - type (string - from valid types above)
   - x (number)
   - y (number)

   FRAME, RECTANGLE, ELLIPSE, GROUP:
   - width (number)
   - height (number)

   TEXT:
   - width (number)
   - height (number)
   - characters (string) - THE ACTUAL TEXT CONTENT
   - fontSize (number)
   - fontName: { "family": "Inter", "style": "Regular" }
   - textAlignHorizontal: "LEFT" | "CENTER" | "RIGHT" | "JUSTIFIED"
   - textAlignVertical: "TOP" | "CENTER" | "BOTTOM"
   - lineHeight: { "unit": "AUTO" } or { "unit": "PIXELS", "value": 24 } or { "unit": "PERCENT", "value": 150 }
   - fills (array)

   VECTOR (for custom shapes ONLY — NOT for icons):
   - width (number)
   - height (number)
   - fills (array)
   - strokes (array)
   - strokeWeight (number)
   - strokeCap: "NONE" | "ROUND" | "SQUARE"
   - strokeJoin: "MITER" | "BEVEL" | "ROUND"
   - vectorPaths (array) - SVG path data

4. Common Visual Properties (optional, apply to ALL nodes):
   - opacity: number (0-1, default 1). Use for subtle overlays, disabled states, decorative elements.
   - visible: boolean (default true)
   - rotation: number (degrees, e.g., 45 for 45 clockwise)
   - blendMode: "NORMAL" | "MULTIPLY" | "SCREEN" | "OVERLAY" | "DARKEN" | "LIGHTEN" (default "NORMAL")

5. Fills Structure (for colors/backgrounds):

   Solid fill:
```json
"fills": [
  {
    "type": "SOLID",
    "visible": true,
    "opacity": 1,
    "blendMode": "NORMAL",
    "color": { "r": 1, "g": 1, "b": 1 }
  }
]
```

   Gradient fill (for hero sections, CTAs, decorative backgrounds):
```json
"fills": [
  {
    "type": "GRADIENT_LINEAR",
    "visible": true,
    "opacity": 1,
    "gradientStops": [
      { "position": 0, "color": { "r": 0.09, "g": 0.46, "b": 0.95, "a": 1 } },
      { "position": 1, "color": { "r": 0.56, "g": 0.24, "b": 0.94, "a": 1 } }
    ]
  }
]
```
   Valid gradient types: "GRADIENT_LINEAR" | "GRADIENT_RADIAL" | "GRADIENT_ANGULAR" | "GRADIENT_DIAMOND"
   Each stop needs: position (0-1) and color with alpha (r, g, b, a all 0-1).

   Image fill (for background images, photos, placeholders):
```json
"fills": [
  {
    "type": "IMAGE",
    "scaleMode": "FILL",
    "imageUrl": "https://example.com/photo.jpg"
  }
]
```
   Valid scaleMode: "FILL" | "FIT" | "CROP" | "TILE"

6. Strokes Structure (for borders):
```json
"strokes": [
  {
    "type": "SOLID",
    "visible": true,
    "opacity": 1,
    "blendMode": "NORMAL",
    "color": { "r": 0.8, "g": 0.8, "b": 0.8 }
  }
],
"strokeWeight": 1,
"strokeAlign": "INSIDE"
```
   Additional stroke properties (optional):
   - strokeAlign: "INSIDE" | "CENTER" | "OUTSIDE"
   - dashPattern: [10, 5] (array of numbers — e.g., [10, 5] = 10px dash, 5px gap; [4, 4] = dotted)
   - strokeCap: "NONE" | "ROUND" | "SQUARE" | "ARROW_LINES" | "ARROW_EQUILATERAL"
   - strokeJoin: "MITER" | "BEVEL" | "ROUND"

7. Corner Radius:
   - Single value: "cornerRadius": 8
   - Individual corners:
     "topLeftRadius": 8,
     "topRightRadius": 8,
     "bottomLeftRadius": 0,
     "bottomRightRadius": 0
   - Smooth corners (iOS-style): "cornerSmoothing": 0.6 (value 0-1, 0.6 matches iOS)

8. Auto-Layout (for FRAME nodes):
```json
"layoutMode": "VERTICAL",
"primaryAxisSizingMode": "AUTO",
"counterAxisSizingMode": "FIXED",
"primaryAxisAlignItems": "MIN",
"counterAxisAlignItems": "CENTER",
"itemSpacing": 16,
"paddingTop": 20,
"paddingRight": 20,
"paddingBottom": 20,
"paddingLeft": 20
```

   Valid values:
   - layoutMode: "NONE" | "HORIZONTAL" | "VERTICAL"
   - primaryAxisSizingMode: "FIXED" | "AUTO"
   - counterAxisSizingMode: "FIXED" | "AUTO"
   - primaryAxisAlignItems: "MIN" | "CENTER" | "MAX" | "SPACE_BETWEEN"
   - counterAxisAlignItems: "MIN" | "CENTER" | "MAX" | "BASELINE"

   Wrap layout (for responsive grids):
   - layoutWrap: "NO_WRAP" | "WRAP" (WRAP enables flex-wrap — children flow to next row/column when full)
   - counterAxisSpacing: number (gap between wrapped rows/columns, only used with layoutWrap: "WRAP")

   Frame overflow:
   - clipsContent: boolean (true = clip children that overflow frame bounds; default true for frames)

   Layout Child Properties (apply to CHILDREN inside an auto-layout frame):
   These go on the child nodes, NOT the parent frame:
   - layoutAlign: "INHERIT" | "STRETCH" (STRETCH = child fills the cross-axis of parent)
   - layoutGrow: 0 | 1 (1 = child fills remaining space along primary axis, like CSS flex-grow: 1)
   - layoutPositioning: "AUTO" | "ABSOLUTE" (ABSOLUTE = child is positioned freely with x/y inside auto-layout parent, removed from flow)

   Example: Two-column layout (sidebar + content):

```json
  "type": "FRAME",
  "width": 1440,
  "height": 900,
  "layoutMode": "HORIZONTAL",

  "itemSpacing": 0,
  "children": [
    {
      "name": "Sidebar",

      "primaryAxisSizingMode": "FIXED",

      "fills": [{ "type": "SOLID", "visible": true, "opacity": 1, "blendMode": "NORMAL", "color": { "r": 0.96, "g": 0.96, "b": 0.97 } }],
      "children": []
    },
    {
      "name": "Main Content",
      "type": "FRAME",
      "x": 0, "y": 0,
      "width": 1160,
      "height": 900,

```

9. Effects (shadows, blurs):
```json
"effects": [
  {
    "type": "DROP_SHADOW",
    "visible": true,
    "radius": 10,
    "color": { "r": 0, "g": 0, "b": 0, "a": 0.25 },
    "offset": { "x": 0, "y": 4 },
    "spread": 0,
    "blendMode": "NORMAL"
  }
]
```

   Valid effect types: "DROP_SHADOW" | "INNER_SHADOW" | "LAYER_BLUR" | "BACKGROUND_BLUR"

   IMPORTANT: Do NOT use "BORDER" as an effect type — it does NOT exist in Figma.
   For borders, use `strokes` (see section 6 above), NOT effects.

   Shadow-specific: showShadowBehindNode: boolean (renders shadow behind the node for DROP_SHADOW)
   Blur-only example: { "type": "LAYER_BLUR", "visible": true, "radius": 10 }

10. TEXT Node — Full Properties:

    Required properties (always include):
    - characters (string) — the actual text content
    - fontSize (number)
    - fontName: { "family": "Inter", "style": "Regular" }
    - textAlignHorizontal: "LEFT" | "CENTER" | "RIGHT" | "JUSTIFIED"
    - textAlignVertical: "TOP" | "CENTER" | "BOTTOM"
    - lineHeight: { "unit": "AUTO" } or { "unit": "PIXELS", "value": 24 } or { "unit": "PERCENT", "value": 150 }
    - fills (array — text color)

    Optional text properties (use to enhance designs):
    - textAutoResize: "NONE" | "HEIGHT" | "WIDTH_AND_HEIGHT" | "TRUNCATE"
      ("HEIGHT" = fixed width, auto height — best for body text in auto-layout)
      ("WIDTH_AND_HEIGHT" = auto size both — best for labels, buttons, tags)
      ("TRUNCATE" = fixed size with ellipsis)
    - textDecoration: "NONE" | "UNDERLINE" | "STRIKETHROUGH"
    - textCase: "ORIGINAL" | "UPPER" | "LOWER" | "TITLE"
    - letterSpacing: { "unit": "PIXELS", "value": 0.5 } or { "unit": "PERCENT", "value": 5 }
    - paragraphSpacing: number (space between paragraphs in px)
    - textTruncation: "DISABLED" | "ENDING" (show "..." when text overflows)
    - maxLines: number (max lines before truncation, use with textTruncation: "ENDING")
    - hyperlink: { "type": "URL", "value": "https://example.com" }

    Mixed text styling (different fonts/colors/sizes in one text node):
```json
{
  "type": "TEXT",
  "name": "Rich Text",
  "characters": "Hello World",
  "fontSize": 16,
  "fontName": { "family": "Inter", "style": "Regular" },
  "textAlignHorizontal": "LEFT",
  "textAlignVertical": "TOP",
  "lineHeight": { "unit": "AUTO" },
  "fills": [{ "type": "SOLID", "visible": true, "opacity": 1, "blendMode": "NORMAL", "color": { "r": 0.1, "g": 0.1, "b": 0.1 } }],
  "textSegments": [
    {
      "start": 0,
      "end": 5,
      "fontName": { "family": "Inter", "style": "Bold" },
      "fontSize": 16,
      "fills": [{ "type": "SOLID", "color": { "r": 0.09, "g": 0.46, "b": 0.95 } }]
    },
    {
      "start": 6,
      "end": 11,
      "fontName": { "family": "Inter", "style": "Regular" },
      "fontSize": 16
    }
  ]
}
```

11. Font Families: Use only these fonts that exist in Figma:
    - "Inter" (styles: "Regular", "Medium", "Semi Bold", "Bold")
    - "Roboto" (styles: "Regular", "Medium", "Bold")
    - "SF Pro" (styles: "Regular", "Medium", "Semibold", "Bold")
    - "Arial" (styles: "Regular", "Bold")
    Do NOT use fonts like "Poppins" unless explicitly available.

12. Shape-Specific Properties:

    ELLIPSE — arcs, donuts, pie slices:
    - arcData: { "startingAngle": 0, "endingAngle": 4.71, "innerRadius": 0.5 }
      Angles in radians (full circle = 6.283). innerRadius 0-1 (0 = full circle, 0.5 = donut).
      Use for: progress rings, pie charts, donut charts.

    POLYGON — regular polygons:
    - pointCount: number (minimum 3; 3 = triangle, 6 = hexagon, 8 = octagon)

    STAR — star shapes:
    - pointCount: number (number of star points, minimum 3)
    - innerRadius: number (0-1, ratio of inner to outer radius; 0.5 = classic star)

13. Multiple Pages: Return an array with multiple FRAME objects. Each top-level FRAME represents a page/screen:
```json
[
  {
    "name": "Login Page",
    "type": "FRAME",
    "x": 0,
    "y": 0,
    "width": 1920,
    "height": 1080,
    ...
  },
  {
    "name": "Dashboard Page",
    "type": "FRAME",
    "x": 0,
    "y": 0,
    "width": 1920,
    "height": 1080,
    ...
  }
]
```

14. Common Screen Sizes:
    - Desktop: 1920 x 1080 or 1440 x 900
    - Mobile: 390 x 844 (iPhone 14/15)
    - Tablet: 820 x 1180

15. ICONS — USE ICONIFY API

When you need an icon in the design, use the `searchIcons` and `getIconUrl` tools, then place the icon as a RECTANGLE with an IMAGE fill. See the separate icon instructions for full details.

Do NOT use VECTOR nodes with vectorPaths for icons.

Quick example:
```json
{
  "name": "Home Icon",
  "type": "RECTANGLE",
  "x": 0,
  "y": 0,
  "width": 24,
  "height": 24,
  "fills": [
    {
      "type": "IMAGE",
      "scaleMode": "FILL",
      "imageUrl": "https://api.iconify.design/solar/home-bold.svg"
    }
  ]
}
```
